# AI 啟動 JARVIS 模式 — 用 Claude Code 指揮真實工作

## Part 3 : Claude Code

> 不是學如何寫程式，而是學會「指揮」AI 完成真實工作任務

---

### 3-1 CLAUDE.md — 給 AI 的工作說明書

CLAUDE.md 是放在資料夾根目錄的一個純文字檔案。它的作用只有一個：讓 Claude Code 每次啟動時，都能快速讀懂這個專案的背景與規範。

還記得 Part 1 提到的「無狀態（Stateless）」嗎？Claude Code 每次啟動都是全新的開始，不記得上次你說過什麼。CLAUDE.md 就是解法——把你希望 AI 永遠記住的事，寫死在一個它每次都會讀的地方。

你可以在 CLAUDE.md 裡寫：

| 內容類型 | 範例 |
|---------|------|
| 專案背景 | 這個資料夾存放公司季報的 Markdown 草稿 |
| 語言與風格 | 繁體中文，正式語氣，不使用縮寫 |
| 輸出規範 | 所有文件用 H2 作為最高層標題，不使用 Emoji |
| 禁止事項 | 不要修改 archive/ 資料夾內的任何檔案 |
| 常用工具 | 使用 Marp 格式製作投影片 |

---

`/init` 會讓 Claude Code 掃描你的工作資料夾，自動生成一份 `CLAUDE.md` 草稿——包含它對這個資料夾的理解：檔案結構、可能的用途、建議的工作規範。

一份實際的 CLAUDE.md 看起來像這樣：

```markdown
# 工作環境說明

## 專案背景
公司週報與客戶簡報的製作資料夾。
所有對外文件需經主管審閱後才能定稿。

## 輸出規範
- 語言：繁體中文，不摻英文術語（需要時加括號標注）
- 投影片：Marp 格式，每頁不超過 5 個重點
- 文件：H2 為最高標題層級

## 禁止動作
- 不修改 final/ 資料夾內的任何檔案
- 不刪除任何超過 30 天的歷史版本
```

> CLAUDE.md 可以隨時修改——直接用 VS Code 開啟編輯即可。它是活的文件，專案跑著跑著，你會不斷發現哪些規則值得加進去。

---

#### CLAUDE.md 的兩個層級

CLAUDE.md 有兩種放置位置，作用範圍不同：

| 層級 | 路徑 | 作用範圍 |
|-----|------|---------|
| 專案級 | 工作資料夾根目錄 | 只影響該專案 |
| 全域級 | 使用者家目錄下的 `.claude/` | 對所有專案生效 |

全域 CLAUDE.md 的路徑：

- Windows：`C:\Users\<使用者名稱>\.claude\CLAUDE.md`
- Mac / Linux：`~/.claude/CLAUDE.md`

> 全域設定適合放「不管在哪個專案都成立」的規範——例如慣用語言、回覆風格、永遠禁止的動作。專案級設定會覆蓋全域設定，讓每個資料夾可以有自己的例外。

### 3-2 用說話的方式下指令

Claude Code 不需要特殊語法。你怎麼對同事說，就怎麼對它說。
但「說話的方式」有好有壞。Claude Code 的執行品質，高度取決於你的指令品質——不是更長，而是更清楚。

一個好指令，通常包含三件事：

```
目標（你要什麼結果）
+
素材（它應該看哪些資料）
+
規範（輸出的格式或限制）
```

這不是公式，而是一種思考習慣。

---

| 情境 | 不夠清楚的指令 | 更好的指令 |
|-----|-------------|-----------|
| 整理會議記錄 | 「幫我整理這份記錄」 | 「讀取 meeting_0606.md，整理成三個部分：決議事項、待辦清單（含負責人）、待確認問題，繁體中文，存回同一個檔案」 |
| 生成簡報 | 「做一份簡報」 | 「用 Marp 製作一份 8 頁投影片，主題是 Q2 業績回顧，讀取 q2_data.csv 作為數據依據，語氣正式，存為 q2_review.md」 |
| 重新命名檔案 | 「整理一下這個資料夾」 | 「掃描 reports/ 資料夾，把所有 .md 檔案重新命名為 YYYYMMDD_原始檔名，日期從檔案第一行標題讀取」 |
| 起草郵件 | 「寫一封信給客戶」 | 「根據 proposal.md 的內容，寫一封跟進信，說明已收到意見並將於三個工作天內回覆，語氣正式但友善，繁體中文」 |

> 指令寫得模糊，Claude Code 會猜——猜的結果可能正確，也可能需要你反覆修正。把一分鐘花在寫清楚指令上，通常比花十分鐘修正錯誤的輸出划算。

### 3-3 掌握你的夥伴正在做什麼

#### 主對話區：動態指示器

任務執行中，對話區會出現類似這樣的文字：

```
⠸ Reading… (8s · ↑2k tokens)
```

格式是：動作 + 已花時間 + 消耗的 token 數量
這代表 Claude Code 現在「正在讀取某個檔案」，已經跑了 8 秒，用了約 2,000 個 tokens。

#### 畫面底部：狀態列

按 `Shift + Tab` 可循環切換三種執行模式：

| 模式 | 畫面顯示 | 說明 |
|-----|---------|------|
| 手動確認 | （無標示）| 每個操作都需要你逐一批准 |
| 自動模式 | `⏵⏵ auto mode on` | 所有操作自動執行，不再詢問 |
| 接受編輯 | `⏵⏵ accept edits on` | 僅檔案編輯自動執行，終端指令仍需確認 |

剛開始使用建議維持手動確認，熟悉 Claude Code 的行為後再視情況開啟自動模式。

---

#### 常見的動作狀態

| 畫面顯示 | Claude Code 正在做的事 | 你需要做什麼 |
|---------|----------------------|------------|
| `Thinking…` | 模型正在推理、規劃下一步 | 等待 |
| `Reading…` | 讀取某個檔案內容 | 等待 |
| `Writing…` | 將內容寫入檔案 | 等待 |
| `Editing…` | 修改現有檔案的部分內容 | 等待 |
| `Bashing…` / `Running…` | 執行系統指令 | 等待，偶爾需確認權限 |
| `Searching…` | 在網路上搜尋資訊 | 等待 |
| `Fetching…` | 抓取某個網頁內容 | 等待 |
| `Listing…` | 瀏覽資料夾結構 | 等待 |
| `Grepping…` | 在檔案中搜尋特定文字 | 等待 |
| `Spawning agent…` / `Tasking…` | 啟動子任務處理複雜工作 | 等待（時間較長）|

### 3-4 三個判斷原則

1. 它還在動嗎？時間數字持續增加、spinner 還在轉，就代表它在跑。不需要做任何事。

2. 時間是否異常長？一般操作幾秒到幾分鐘不等。如果超過你預期很多，可以連按兩次 `Escape` 中斷，再重新描述任務。

3. 出現提示了嗎？某些操作 Claude Code 會暫停並詢問你是否允許。這時需要你按鍵回應——這不是當掉，是它在等你。

### 3-5 權限系統 — 你的信任邊界

Claude Code 在執行某些敏感操作前，會暫停並詢問你的許可：

```
Claude Code wants to execute a bash command:
  git push origin main

1. Yes
2. Yes, always
3. No
4. Explain
```

這不是障礙，而是設計。它讓你知道 AI 正在做什麼，並保持最終控制權。

| 選項 | 意義 | 適合時機 |
|-----|------|---------|
| Yes（本次同意）| 這次允許，下次再問 | 第一次嘗試某個操作 |
| Yes, always（永遠同意）| 這類操作以後不再詢問 | 你已確認這個操作安全 |
| No（拒絕）| 不執行，Claude Code 會尋找替代方案 | 不確定這個操作是否正確 |
| Explain（說明）| 讓 Claude Code 解釋它為什麼要這樣做 | 看不懂它在做什麼時 |

---

你可以在 CLAUDE.md 中預先指定信任範圍，減少重複詢問：

```markdown
## 已授權操作（不需每次詢問）
- 讀取和修改當前資料夾內的任何 .md 檔案
- 執行 git add、git commit
- 在 reports/ 資料夾內建立新檔案

## 需要每次確認的操作
- git push（推送到遠端前必須確認）
- 刪除任何檔案
- 修改 final/ 資料夾內的檔案
```

> 剛開始使用時，讓它多問一點沒關係。等你熟悉了它的行為模式，再逐步擴大「不需詢問」的範圍——就像你跟一位新同事建立信任的過程一樣。

### 3-6 五個工作流程

不需要任何程式背景，就能立刻用 Claude Code 完成的實際工作任務。

| 工作流程 | Claude.ai | Claude Code | 關鍵差異 |
|----------|-----------|-------------|---------|
| 錄音稿 → 會議記錄 | 部分可以（需手動貼上、複製輸出）| ✅ | 自動讀檔、直接寫入本地 .md |
| 批次整理資料夾 | ❌ | ✅ | 掃描資料夾、重新命名、移動檔案 |
| CSV → Marp 簡報 | 部分可以（需手動貼上、複製輸出）| ✅ | 直接讀 CSV、存為本地檔案 |
| 起草客戶跟進信 | 部分可以（上傳檔案後）| ✅ | 讀檔＋寫稿＋自動存檔一氣呵成 |
| 建立知識庫索引 | ❌ | ✅ | 批次掃描整個資料夾的多個檔案 |

---

> Claude.ai 在瀏覽器裡運作，碰不到你電腦的檔案系統。
> Claude Code 在你的電腦上運作，可以讀、寫、移動、執行任何本地檔案。

工作流程一：把錄音稿變成結構化會議記錄

```
讀取 meeting_transcript.txt
整理成以下結構：
1. 會議基本資訊（日期、出席者）
2. 決議事項（條列，每條標注負責人與截止日）
3. 待辦清單（按優先級排序）
4. 待確認問題
輸出為 meeting_0606.md，繁體中文
```

工作流程二：批次整理資料夾

```
掃描 inbox/ 資料夾
把所有 .docx 和 .pdf 檔案：
- 重新命名為 YYYYMMDD_原始檔名
- 移入 processed/ 子資料夾（沒有就建立）
- 把處理清單輸出到 log.md
```

---

工作流程三：從數據生成 Marp 簡報

```
讀取 q2_sales.csv
生成一份 Marp 投影片，主題是 Q2 業績回顧：
- 8 頁，每頁不超過 5 個重點
- 包含關鍵數字和趨勢說明
- 語氣：給主管的報告，正式但不冗長
- 存為 q2_review.md
```

工作流程四：起草客戶跟進信

```
讀取 proposal.md，了解方案的核心優勢
起草一封跟進信：
- 表示理解，不施壓
- 簡要重申方案的兩個核心優勢
- 提議下週安排 30 分鐘電話釐清疑問
- 語氣：誠懇、專業，繁體中文，200 字以內
存為 follow_up_draft.md
執行 git add follow_up_draft.md
執行 git commit -m "draft: client follow-up email"
```

---

工作流程五：建立個人知識庫索引

```
讀取 notes/ 資料夾內所有的 .md 檔案
幫我：
1. 列出所有出現超過 3 次的主題關鍵字
2. 為每個關鍵字建立一個索引條目，列出相關筆記與頁碼
3. 生成一份 index.md 作為整個知識庫的入口
```

> 這五個工作流程的共同特點：你只需要描述你要什麼結果——不需要知道 Claude Code 用了什麼工具、執行了幾個步驟。結果導向，是 Agentic 工作方式最根本的轉變。

### 3-7 常用 Slash 指令速查

在對話框輸入 `/` 開頭的指令，可以快速控制 Claude Code 的行為：

| 指令 | 功能 | 使用時機 |
|-----|------|---------|
| `/init` | 掃描資料夾，生成 CLAUDE.md | 第一次使用新資料夾時 |
| `/help` | 顯示所有可用指令 | 忘記指令時查詢 |
| `/status` | 開啟設定面板，查看帳號狀態或調整配置 | 確認目前工作設定、切換執行模式時 |
| `/clear` | 清除當前對話記錄，重新開始 | 開始新任務、避免舊對話干擾 |
| `/compact` | 壓縮對話歷史，節省 token | 對話很長、AI 開始跑偏時 |
| `/resume` | 列出歷史 session，選擇後接續上次對話 | 關掉再開，想繼續上次未完成的任務 |
| `/model` | 切換使用的 AI 模型 | 需要更快速度或更強推理時 |
| `/review` | 審查 Claude Code 對檔案的修改 | 確認 AI 動過哪些東西 |
| `/exit` | 結束 Claude Code | 工作完成、關閉 session |

---

> `/clear` 和 `/compact` 是最常用的兩個——當對話累積過長、AI 的回答開始偏離主題，這兩個指令能把它「重新拉回來」。`/clear` 是全部清空重來，`/compact` 是在保留重點摘要的前提下壓縮，視情況選擇。

### 3-8 自訂指令（User Commands）

把常用的工作流程存成一個 `.md` 檔，放在以下位置，就能用 `/檔案名稱` 一鍵呼叫：

| 範圍 | Windows | Mac / Linux |
|-----|---------|-------------|
| 全域（所有專案） | `C:\Users\<名稱>\.claude\commands\` | `~/.claude/commands/` |
| 專案（只限當前） | `.claude\commands\`（放在資料夾根目錄） | `.claude/commands/` |

```
/週報  →  執行你預先定義好的「生成本週工作摘要」流程
```

---

### 3-9 Skills

Skills 是 Claude Code 提供的專項能力模組（例如程式碼審查、安全性分析）。
在對話框輸入 `/` 後，下拉選單會列出目前可用的所有 Skills，直接點選即可啟用。

> 自訂指令讓你把重複的工作流程變成一個指令；Skills 讓 AI 在特定任務上展現更深的專業能力。
