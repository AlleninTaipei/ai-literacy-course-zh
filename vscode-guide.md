# Visual Studio Code (VS Code)

## 文字編輯器, 給職場工作者的 VS Code 入門指南

## 前言

VS Code 是微軟推出的免費文字編輯器, 雖然最初是為程式開發設計, 但因為輕量, 跨平台, 且可透過擴充套件擴展功能, 現在也很適合用來撰寫 Markdown 文件, 編輯設定檔, 管理筆記, 以及搭配 Git 做版本控制. 對於已經接觸過前面幾份文件(Markdown, Marp, Git & GitHub, Claude Code)的職場工作者來說, VS Code 正是把這些工具串接在一起的共同工作環境.

## 什麼是 VS Code

Visual Studio Code 是一套免費, 開放原始碼, 跨平台(Windows, Mac, Linux)的文字編輯器. 它本身功能精簡, 但透過安裝擴充套件(extensions), 可以擴展出接近完整開發環境的能力, 例如語法高亮, 即時預覽, 版本控制整合等.

## 介面總覽

| 區域 | 說明 |
| --- | --- |
| 活動列(Activity Bar) | 畫面最左側的圖示列, 用於切換檔案總管, 搜尋, 版本控制, 擴充套件等主要功能 |
| 檔案總管(Explorer) | 顯示目前資料夾中的檔案與子資料夾結構 |
| 編輯區 | 顯示與編輯目前開啟的檔案內容, 可同時開啟多個分頁或分割視窗 |
| 終端機(Terminal) | 內建的指令列視窗, 可在不離開編輯器的情況下執行指令 |
| 狀態列 | 畫面最下方, 顯示目前檔案格式, 編碼, 行數位置等資訊 |

## 基本操作

開始使用時, 建議用 "Open Folder" 開啟一整個工作資料夾, 而不是只開單一檔案, 這樣才能使用檔案總管, 整個資料夾搜尋, 以及版本控制等完整功能.

開啟資料夾後, 可以在檔案總管中點擊檔案開啟, 在編輯區同時開啟多個分頁, 也可以把編輯區分割成左右或上下兩個區塊, 方便同時對照兩份文件內容.

## 命令面板 Command Palette

按下 `Ctrl+Shift+P` 可以開啟命令面板, 用文字搜尋並執行幾乎所有 VS Code 的功能, 不需要記住每個功能藏在哪一層選單裡, 是最有效率的操作方式之一. 多數快捷鍵在 Mac 上會把 `Ctrl` 換成 `Cmd`.

## 擴充套件 Extensions

點擊活動列中的擴充套件圖示, 或用命令面板搜尋 "Extensions: Install Extensions", 即可在線上市集搜尋並安裝套件. 以下是與前面文件主題相關, 對職場工作者實用的延伸套件.

| 擴充套件 | 用途 |
| --- | --- |
| 內建 Markdown 預覽 | 不需額外安裝, 即可即時預覽 Markdown 文件的排版效果 |
| Marp for VS Code | 撰寫與預覽 Marp 簡報, 並可直接匯出 PDF, PPTX |
| GitLens | 加強內建的 Git 功能, 方便查看每一行內容的修改紀錄 |
| Claude Code(IDE 擴充套件) | 在編輯器內直接與 Claude Code 互動, 不需切換到獨立終端機視窗 |

## 內建 Git 整合

VS Code 內建支援 Git, 不需要額外安裝套件就能在側邊欄看到目前哪些檔案有變更, 並透過按鈕完成 commit, push, pull 等操作, 對於不熟悉指令列的使用者來說, 是入門 Git 的友善方式. 操作的底層概念與前面的 Git & GitHub 文件相同, 只是改用圖形介面點擊完成.

## Markdown 即時預覽

開啟一份 `.md` 檔案後, 按下 `Ctrl+Shift+V` 可以開啟即時預覽視窗, 或按下 `Ctrl+K V` 在旁邊開一個分割視窗同步預覽, 方便邊寫邊確認排版效果, 適合搭配前面介紹過的 Markdown 與 Marp 文件一起使用.

## 主題與外觀

可以在設定中更換顏色主題, 圖示主題, 以及調整字型大小與行距, 對於需要長時間閱讀與編輯文件的使用者, 適當調整外觀有助於降低閱讀疲勞.

## 職場工作者常見應用情境

### 撰寫與維護文件

把會議記錄, SOP, 教學講義等 Markdown 文件統一放在一個工作資料夾中, 用 VS Code 開啟整個資料夾管理, 並利用即時預覽確認排版結果.

### 製作與預覽簡報

搭配 Marp for VS Code 擴充套件, 直接在 VS Code 中撰寫簡報用的 Markdown, 即時預覽投影片效果, 完成後再匯出成 PDF 或 PPTX.

### 文件版本控制

利用內建的 Git 圖形介面, 在不需要輸入指令的情況下, 完成文件修改的版本紀錄與同步, 與團隊成員協作時更容易追蹤異動.

### 搭配 Claude Code 處理日常工作

安裝 Claude Code 的 VS Code 擴充套件後, 可以在編輯器內直接請 Claude 協助建立, 修改, 或整理文件, 不需要另外切換到獨立的終端機視窗.

## 使用時的基本注意事項

1. 建議用 "Open Folder" 開啟整個工作資料夾, 只開單一檔案會無法使用檔案總管與整個資料夾搜尋等功能.
2. 不需要的擴充套件盡量不要安裝, 過多擴充套件會拖慢啟動與編輯速度.
3. 修改檔案後留意是否已經儲存, 分頁標題若顯示一個圓點, 代表內容尚未儲存.
4. 內建終端機預設使用的命令列環境(例如 PowerShell 或 bash)可能與平常習慣不同, 留意指令語法上的差異.
5. 同時開啟多個視窗或工作區時, 切換前養成隨手存檔的習慣, 避免變更遺失.

| 即時預覽 | 一邊編輯 Markdown 或簡報, 一邊同步看到排版結果 |
| 內建終端機 | 不離開編輯器即可使用的指令列視窗 |

## 結語

VS Code 本身學習門檻不高, 只要熟悉開啟工作資料夾, 命令面板, 以及搭配 Markdown 預覽與內建 Git 等基本操作, 就能大幅提升日常文件處理的效率. 建議把前面幾份文件(Markdown, Marp, Git & GitHub, Claude Code)放進同一個 VS Code 工作資料夾中練習, 體會這些工具如何串接成一套完整的工作流程.
