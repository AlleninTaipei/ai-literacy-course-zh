# Markdown

## 文件排版標記語言, 給職場工作者的 Markdown 入門指南

## 前言

Markdown 是一種輕量級的標記語言, 用最少的符號就能完成排版, 同時保留純文字的可讀性. 對於需要撰寫會議記錄, 專案文件, 技術筆記或知識庫的職場工作者來說, 學會 Markdown 可以大幅提升文件產出的效率與一致性.

本文件從基礎語法開始, 逐步介紹 Markdown 常用的標記方式, 並提供辦公場景中的實際應用範例.

## 什麼是 Markdown

Markdown 由 John Gruber 於 2004 年提出, 目的是讓人可以用簡單易讀的純文字格式寫作, 再透過轉換工具產生 HTML 或其他格式的文件. 常見的使用場景包括:

- GitHub, GitLab 等程式碼平台的 README 與 Issue
- Notion, HackMD, Obsidian 等筆記與知識庫工具
- Slack, Teams 等通訊軟體的訊息格式
- 部落格與靜態網站產生器, 例如 Hugo, Jekyll, Hexo

## 標題 Heading

使用 `#` 來建立標題, `#` 的數量代表標題層級, 最多支援六層.

```
# 一級標題
## 二級標題
### 三級標題
#### 四級標題
##### 五級標題
###### 六級標題
```

建議, 同一份文件中標題層級不要跳階使用, 例如一級標題後直接接三級標題, 這樣會讓文件結構不清楚.

## 段落與換行

段落之間只要保留一行空白即可分段. 若要在同一段落內換行, 需要在行尾加上兩個空白字元, 或使用 `<br>` 標籤.

```
這是第一行(行尾有兩個空白字元)
這是第二行
```

## 強調語法

| 效果 | 語法 | 說明 |
| --- | --- | --- |
| 斜體 | `*文字*` 或 `_文字_` | 文字會以斜體顯示 |
| 粗體 | `**文字**` 或 `__文字__` | 文字會以粗體顯示 |
| 粗斜體 | `***文字***` | 文字會同時呈現粗體與斜體 |
| 刪除線 | `~~文字~~` | 文字中間會加上一條刪除線 |

## 清單

### 項目清單(無序)

語法用 `-`, `*`, 或 `+` 開頭, 後面加一個空白.

```
- 第一項
- 第二項
  - 子項目一
  - 子項目二
- 第三項
```

### 編號清單(有序)

```
1. 第一步
2. 第二步
3. 第三步
```

提醒, Markdown 編號清單的數字大多會自動排序, 即使全部寫成 `1.`, 渲染後仍會自動編號為 1, 2, 3.

## 連結與圖片

```
[顯示文字](網址 "選填的提示文字")
![替代文字](圖片網址 "選填的提示文字")
```

範例:

```
[Anthropic 官網](https://www.anthropic.com)
![公司 Logo](./images/logo.png)
```

## 程式碼

行內程式碼使用單個反引號包住文字, 例如 `這是行內程式碼`.

區塊程式碼使用三個反引號圍住多行內容, 並可以在開頭反引號後加上語言名稱以啟用語法高亮, 例如:

````
```python
def hello():
    print("Hello, Markdown")
```
````

## 引言區塊 Blockquote

使用 `>` 開頭可以建立引言區塊, 也可以多層巢狀.

```
> 這是引言.
>> 這是引言中的引言.
```

## 表格

```
| 欄位一 | 欄位二 | 欄位三 |
| --- | --- | --- |
| 內容 A | 內容 B | 內容 C |
| 內容 D | 內容 E | 內容 F |
```

對齊方式可以在分隔列使用冒號控制, 例如 `:---` 靠左, `:---:` 置中, `---:` 靠右.

## 分隔線

連續三個或以上的 `-`, `*`, 或 `_` 可以建立一條水平分隔線, 常用於區分章節.

```
---
```

## 待辦清單 Task List

許多平台(GitHub, Notion, HackMD)支援待辦清單語法.

```
- [ ] 尚未完成的任務
- [x] 已經完成的任務
```

## 跳脫字元

若想顯示 Markdown 語法符號本身而不被解讀為格式, 在符號前加上反斜線 `\`.

```
\*這段文字不會變成斜體\*
```

## 職場工作者常見應用情境

### 會議記錄

```
# 2026-06-21 專案進度會議

## 出席人員
- 王小明(主持人)
- 林小美

## 討論事項
1. 上週進度回顧
2. 本週待辦事項
3. 風險與待解決問題

## 待辦清單
- [ ] 王小明: 完成需求文件
- [ ] 林小美: 確認供應商報價
```

### 專案說明文件 README

````
```
# 專案名稱

## 簡介
這個專案的目的是...

## 安裝方式
\`\`\`
npm install
\`\`\`

## 使用方式
說明如何執行與設定.
```
````

## 常見錯誤提醒

1. 標題符號 `#` 後面忘記加空白, 導致無法正確渲染.
2. 清單項目與上一行之間沒有空白行, 造成清單無法正確顯示.
3. 表格分隔列 `---` 數量與欄位數量不一致.
4. 中英文字混排時忘記加空白, 在部分平台會影響排版美觀.
5. 圖片或連結網址使用本機路徑, 換到其他環境後路徑會失效.

## 結語

Markdown 的語法不多, 但只要熟練常用的標題, 清單, 連結, 程式碼與表格語法, 就足以應付絕大多數的辦公文件需求. 建議在日常工作中多加練習, 逐步將會議記錄, 專案文件與知識庫筆記都轉換成 Markdown 格式, 長期下來會大幅提升文件管理的效率.
